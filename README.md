# 🇸🇾 بوابة الحكومة الإلكترونية الموحدة - مشروع التخرج 🇸🇾

مرحباً بكم في المستودع البرمجي الرسمي **لبوابة الحكومة الإلكترونية السورية الموحدة**. تم تصميم هذا النظام كنموذج متكامل وعميق تقنياً (Hero Services Architecture) لتقديم الخدمات الحكومية والمصرفية إلكترونياً بشكل يحاكي النظم الحقيقية باستخدام تقنيات الويب الحديثة وفي الوقت الحقيقي (Real-Time).

---

## 🚀 التقنيات المستخدمة (Tech Stack)

### 💻 القسم الأمامي (Frontend):
* **React.js (Vite)**: لبناء واجهة مستخدم فائقة السرعة والاستجابة.
* **Tailwind CSS**: لتصميم واجهات عصرية تتناسب مع الهوية البصرية الرسمية السورية الجديدة.
* **Zustand**: لإدارة الحالة العالمية للموقع (State Management).
* **Socket.io-Client**: للتواصل الفوري وفي الوقت الحقيقي مع الخادم.
* **PWA (Progressive Web App)**: لدعم التثبيت على الهواتف كبرنامج أصلي والتخزين المؤقت في وضع عدم الاتصال (Offline Caching).

### ⚙️ القسم الخلفي (Backend):
* **Node.js (Express.js)**: البيئة التشغيلية لخدمات الـ API.
* **PostgreSQL**: قاعدة البيانات العلائقية لتخزين السجلات المدنية والعقارية والمالية.
* **Prisma ORM**: للتعامل مع قاعدة البيانات بكل مرونة وكفاءة.
* **Socket.io**: لإرسال الإشعارات والعمليات الفورية.
* **PDFKit & RTL-Arabic**: لتوليد وثائق حكومية رسمية بصيغة PDF قابلة للتحميل ومكتوبة باللغة العربية الصحيحة ومختومة برمز QR ديناميكي للتحقق.

---

## 🛠️ متطلبات التشغيل (Prerequisites)
قبل البدء، يرجى التأكد من تثبيت البرامج التالية:
1. **Node.js** (الإصدار 18 أو أحدث).
2. **PostgreSQL** (مخدم قاعدة بيانات نشط محلياً أو سحابياً).

---

## 🔧 خطوات التثبيت والتشغيل المحلي (Setup Guide)

### 1️⃣ إعداد خادم الخلفية (Backend)
1. انتقل إلى مجلد الخلفية:
   ```bash
   cd backend
   ```
2. قم بتثبيت الحزم البرمجية:
   ```bash
   npm install
   ```
3. قم بإنشاء ملف المتغيرات البيئية `.env` في جذر مجلد `backend` وتكوينه كالتالي:
   ```env
   DATABASE_URL="postgresql://username:password@localhost:5432/eGOV?schema=public"
   JWT_SECRET="supersecretkey"
   PORT=5000
   ```
   *(قم باستبدال `username` و `password` ببيانات مخدم الـ PostgreSQL لديك).*

4. قم بتطبيق هجرات قاعدة البيانات (Prisma Migrations):
   ```bash
   npx prisma migrate dev
   ```
5. قم بتغذية قاعدة البيانات بالبيانات الحكومية التجريبية والمخطط الشامل (Database Seeding):
   ```bash
   npx prisma db seed
   ```
6. قم بتشغيل الخادم في وضع التطوير:
   ```bash
   npm run dev
   ```

### 2️⃣ إعداد الواجهات الأمامية (Frontend)
1. افتح نافذة طرفية جديدة وانتقل إلى مجلد الواجهة:
   ```bash
   cd frontend
   ```
2. قم بتثبيت الحزم البرمجية:
   ```bash
   npm install
   ```
3. قم بتشغيل خادم التطوير الأمامي:
   ```bash
   npm run dev
   ```
4. لبناء تطبيق الـ PWA والتخزين المؤقت للإنتاج:
   ```bash
   npm run build
   ```

## 🌟 سيناريوهات العرض المميزة (Demo Scenarios to Show the Jury)

### 🥇 السيناريو الأول: فك الرهن العقاري الآلي (Mortgage Release Integration)
1. قم بتسجيل الدخول كـ **أحمد السوري** (`1234567890`).
2. توجه إلى **السجل العقاري**، ستلاحظ وجود عقارين باسمك: شقة في أبو رمانة (سليمة) ومحل تجاري في **المزة** عليه **إشارة رهن نشطة** باللون الأحمر لصالح المصرف التجاري.
3. حاول تحميل "بيان قيد عقاري" للمحل؛ سيقوم النظام **بحظرك فوراً** ويظهر رسالة تفيد بوجود ضرائب عقارية غير مدفوعة بقيمة `75,000 ل.س`.
4. توجه إلى **المالية والضرائب** وسدد الضريبة العقارية أولاً عبر إشعار بنكي (مثلاً من المصرف المركزي).
5. بعد السداد، ستتمكن من طلب السند بنجاح. لكن السند لا يزال يظهر **إشارة رهن نشطة**.
6. لتصفية الرهن، اذهب إلى قسم المالية وستجد تكليفاً برقم **"تسديد القرض العقاري - فك رهن محضر المزة"** بقيمة `2,000,000 ل.س`.
7. قم بالسداد إلكترونياً برقم الحوالة البنكية؛ وسيقوم النظام **تلقائياً بفك الرهن** في قاعدة البيانات العقارية الموحدة!
8. عد إلى السجل العقاري لتحميل بيان القيد، ستجده قد تحول إلى اللون الأخضر ومكتوب عليه رسمياً: **"خالٍ من أي إشارة رهن"**.

### 🥈 السيناريو الثاني: السجل العدلي الفوري (Criminal Record Clearance)
1. أثناء تسجيل دخولك كـ **أحمد السوري**، توجه إلى **السجل العدلي - غير محكوم**.
2. اكتب الغرض من الوثيقة (مثال: "وزارة التربية").
3. سيقوم النظام بعمل تدقيق جنائي آلي في الخلفية (Automated Background Check) ويصدر الوثيقة فوراً باللون الأخضر المميز مع رمز الاستجابة السريعة (QR Code) لأن سجله العدلي سليم تماماً.
4. للتجربة المعاكسة: قم بتسجيل الدخول برقم وطني لشخص لديه سوابق جنائية مسجلة في قاعدة البيانات (مثل ياسين الحلبي `3333333333`)، وحاول طلب الوثيقة؛ سيقوم النظام **برفض الطلب فوراً وبشكل آلي** ويوضح الجرم المسجل بحقه لمنع التزوير.

### 🥉 السيناريو الثالث: تسجيل الولادات المباشر (Birth Workflow)
1. سجل دخولك كـ **أحمد**، وقم بتعبئة نموذج تسجيل مولود جديد مع رفع صورة شهادة المشفى.
2. سجل الخروج ثم ادخل كـ **سارة الموظفة** (`0000000001`).
3. ستجد الطلب معلقاً في طابور العمليات. يمكنك الضغط على "عرض الوثيقة" لرؤية شهادة المشفى المرفوعة، ثم اضغط "موافقة".
4. سيقوم النظام فوراً بـ:
   * توليد رقم وطني جديد للطفل وتحديث القيد المدني للعائلة في الداتابيز.
   * إرسال إشعار فوري وبث مباشر (Socket.io) إلى حساب المواطن أحمد ليظهر له الرقم الوطني الجديد لمولوده.
