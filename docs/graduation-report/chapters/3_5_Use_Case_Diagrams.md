# 3.5 مخططات حالات الاستخدام (Use Case Diagrams)

يعمل مخطط حالات الاستخدام كخريطة بصرية توضح العلاقة بين الجهات الفاعلة (Actors) والخدمات المختلفة التي يوفرها النظام. يركز هذا المخطط على "ماذا" يقدم النظام بدلاً من "كيف".

## المخطط العام للنظام

يوضح المخطط أدناه تداخل الصلاحيات بين المواطن والموظف والبنك داخل **منصة الحكومة الإلكترونية**:

```mermaid
useCaseDiagram
    actor "المواطن" as Citizen
    actor "الموظف الحكومي" as Employee
    actor "النظام البنكي" as Bank

    package "منصة الحكومة الإلكترونية" {
        usecase "تسجيل الدخول / التوثيق" as UC1
        usecase "استخراج إخراج قيد / وثائق" as UC2
        usecase "تقديم طلب تسجيل ولادة" as UC3
        usecase "بدء نقل ملكية مركبة" as UC4
        usecase "قبول/رفض عرض الشراء" as UC5
        usecase "استخراج براءة ذمة" as UC6
        usecase "مراجعة واعتماد الطلبات" as UC7
        usecase "التحقق من الدفع البنكي" as UC8
        usecase "إدارة الحساب والـ PIN" as UC9
        usecase "الاستعلام عن العقارات" as UC10
    }

    Citizen --> UC1
    Citizen --> UC2
    Citizen --> UC3
    Citizen --> UC4
    Citizen --> UC5
    Citizen --> UC6
    Citizen --> UC9
    Citizen --> UC10

    Employee --> UC1
    Employee --> UC7

    Bank --> UC8
    UC4 ..> UC8 : <<include>>
    UC6 ..> UC8 : <<include>>
    UC7 ..> UC3 : <<extend>>
```

## وصف المكونات

1.  **المواطن (Citizen):** هو المحرك الأساسي لأغلب حالات الاستخدام، حيث تبدأ الدورة بطلبه للخدمة (مثل بدء نقل الملكية أو طلب الوثائق).
2.  **الموظف الحكومي (Employee):** يقتصر دوره على حالات الاستخدام الرقابية مثل مراجعة الطلبات واعتمادها نهائياً.
3.  **النظام البنكي (Bank):** يعمل كفاعل ثانوي يدعم حالات الاستخدام التي تتطلب دفعاً مالياً، حيث يقوم النظام بالتحقق من صحة الحوالات قبل السماح للموظف بالاعتماد.
4.  **العلاقات (Relationships):**
    *   **تضمين (Include):** تتطلب عملية نقل الملكية (UC4) بالضرورة التحقق من الدفع (UC8).
    *   **توسيع (Extend):** عملية الاعتماد (UC7) هي خطوة إضافية اختيارية تعتمد على نجاح تقديم الطلب (UC3).

يوفر هذا المخطط رؤية شمولية لكيفية ترابط العمليات الحكومية والمصرفية لخدمة المواطن في بيئة رقمية واحدة.
