# 5.3 تصميم واجهات برمجة التطبيقات (API Design)

تعتمد **منصة الحكومة الإلكترونية** على معمارية **RESTful API** لتنظيم التواصل بين الواجهة الأمامية والخادم. تم تصميم الواجهات لتكون نمطية (Modular)، آمنة، وسهلة الصيانة.

## 1. مبادئ التصميم (Design Principles)
*   **استخدام أفعال الـ HTTP:** تم الالتزام بالمعايير العالمية في تسمية المسارات:
    *   `GET`: للاستعلام عن البيانات (مثل سجلات المواطنين أو المخالفات).
    *   `POST`: لإنشاء موارد جديدة (مثل تسجيل ولادة أو بدء نقل ملكية).
    *   `PUT`: لتحديث الموارد الموجودة (مثل قراءة التنبيهات).
*   **تنسيق البيانات:** يتم تبادل البيانات حصراً بتنسيق **JSON**، مما يضمن التوافقية العالية وسرعة المعالجة.

## 2. هيكلية المسارات (Route Structure)
تم تقسيم الـ APIs إلى وحدات وظيفية تتوافق مع بنية قاعدة البيانات:
*   `/api/auth`: لإدارة عمليات تسجيل الدخول والتوثيق.
*   `/api/civil`: للتعامل مع سجلات النفوس، الولادات، والزواج.
*   `/api/traffic`: لإدارة المركبات وعمليات نقل الملكية.
*   `/api/tax`: للاستعلام عن الديون المالية وبراءات الذمة.
*   `/api/bank`: لتنفيذ العمليات المصرفية والتحقق من الـ PIN.

## 3. الأمان والتوثيق (Security & Middleware)
تخضع كافة المسارات الحساسة لنظام حماية صارم:
*   **برمجية التوثيق (Auth Middleware):** يتم التحقق من وجود رمز JWT صالح في ترويسة الطلب (`Authorization Header`) قبل السماح بالوصول لأي بيانات.
*   **التحقق من الأدوار:** يتم فحص دور المستخدم (CITIZEN أو EMPLOYEE) برمجياً لمنع المواطنين من الوصول إلى مسارات الموظفين.

## 4. نمط الاستجابة والأخطاء (Error Handling)
يعتمد النظام نمطاً موحداً للاستجابة يسهل التعامل معه في الواجهة الأمامية:
*   **النجاح:** يتم إرجاع الكود `200 OK` مع البيانات المطلوبة.
*   **الأخطاء:** يتم إرجاع أكواد مناسبة (`401` للتوثيق، `403` للصلاحيات، `404` لعدم الوجود) مع رسالة خطأ باللغة العربية توضح للمستخدم طبيعة المشكلة.

## 5. واجهات التواصل اللحظي (Socket Events)
إلى جانب الـ REST API، يضم النظام "واجهات أحداث" (Event Endpoints) لإدارة التفاعلات اللحظية:
*   `vehicle_transfer_offer`: إرسال عرض الشراء للمشتري.
*   `new_birth_request`: إرسال تنبيه للموظف بوجود طلب جديد.
*   `new_notification`: إرسال التنبيهات العامة للمواطنين.

يضمن هذا التصميم المتكامل أن تكون المنصة قادرة على التوسع مستقبلاً وإضافة قطاعات حكومية جديدة بسهولة عبر إضافة مسارات (Routes) جديدة تتبع نفس النمط المعماري.
