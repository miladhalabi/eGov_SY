# الشريحة 15: مسار المعاملة (Workflow Logic)

## هندسة "المصافحة الرقمية"
يوضح مخطط النشاط (Activity Diagram) كيف يدير النظام عملية نقل ملكية مركبة:
1.  **التحقق الأولي:** النظام يفحص ديون البائع؛ إذا وجد شيئاً، تتوقف العملية فوراً (Fail-fast).
2.  **الاتصال بالطرف الثاني:** إرسال إشعار للمشتري. إذا رفض، يتم إلغاء المعاملة.
3.  **التوثيق المالي:** انتظار رقم المرجع البنكي والتحقق منه برمجياً.
4.  **الاعتماد البشري:** الموظف يراجع "الصورة الكلية" للمعاملة ويضغط "موافق".

## النتيجة البرمجية
استخدام **Prisma Transactions** يضمن تحديث كافة الجداول (تبديل المالك + تحديث حالة الطلب + إنشاء الإشعارات) كحزمة واحدة، مما يمنع حدوث تضارب في البيانات في حال انقطاع السيرفر.
